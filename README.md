# 27-
Сайт для портфолио

## База знаний эксперта

Страница: `blog.html`

### Добавление через админку (рекомендуется)

1. Настройте Supabase (см. ниже): `js/supabase-config.js` + SQL из `supabase/schema.sql`
2. Создайте пользователя в Supabase Authentication
3. Откройте `admin.html` → вкладка **Подкасты**
4. Заполните форму и нажмите **Опубликовать выпуск**
5. Скопируйте ссылку кнопкой «Скопировать ссылку» и отправляйте в Telegram

Аудио можно загрузить MP3 прямо в админке (хранится в Supabase Storage) или указать путь `audio/file.mp3`.

### Добавление вручную (резерв)

Если Supabase не подключён, выпуски берутся из `js/podcasts.js` (массив `FALLBACK_PODCASTS`).

```js
{
  id: "vypusk-3",
  title: "Заголовок выпуска",
  date: "2026-07-15",
  duration: "8 мин",
  audio: "audio/vypusk-3.mp3",
  description: "Кратко о чём этот подкаст (1–2 предложения).",
  article: "Полный текст выпуска. Можно несколько абзацев через \\n\\n."
}
```

**Прямая ссылка на выпуск** (для Telegram и соцсетей):

```
https://ваш-сайт.github.io/27-/blog.html#vypusk-1
```

Счётчик подключается **только после** нажатия «Принять» в cookie-баннере.

### Шаг 1. Создайте счётчик

1. Откройте [Яндекс.Метрику](https://metrika.yandex.ru/)
2. Войдите под Яндекс-ID (или зарегистрируйтесь)
3. Нажмите **«Добавить счётчик»**
4. Заполните:
   - **Имя:** Галина Оноприенко — сайт
   - **Адрес сайта:** ваш URL на GitHub Pages (например `https://galileya2008-byte.github.io/27-/`)
5. Включите опции:
   - ✅ Вебвизор
   - ✅ Карта кликов
   - ✅ Отслеживание хеша в адресной строке (если есть)
6. Примите условия и нажмите **«Создать счётчик»**

### Шаг 2. Скопируйте номер счётчика

На странице счётчика найдите **номер** (только цифры, например `104825673`).

### Шаг 3. Вставьте номер в сайт

В файле `js/metrika-config.js`:

```js
window.SITE_METRIKA_ID = 104825673;
```

(без кавычек, только цифры)

### Шаг 4. Проверьте работу

1. Опубликуйте сайт (push в GitHub)
2. Откройте сайт в режиме инкогнито
3. Нажмите **«Принять»** в cookie-баннере
4. В Метрике: **Настройки → Проверка счётчика** — должна появиться отметка «Данные поступают»

### Шаг 4. Где смотреть статистику

- [metrika.yandex.ru](https://metrika.yandex.ru/) → ваш счётчик
- **Отчёты → Стандартные отчёты → Посетители** — кто и когда заходил
- **Вебвизор** — запись сессий (как человек листает страницу)
- **Карта кликов** — куда нажимают

### Цели (JavaScript-события)

В Метрике: **Настройки → Цели → Добавить цель** → тип **JavaScript-событие**.
Создайте 4 цели с такими идентификаторами:

| Идентификатор | Когда срабатывает |
|---|---|
| `telegram_click` | клик по ссылке в Telegram (`telegram.me/galina1901`) |
| `interactive_pay` | клик «Забронировать интерактив» |
| `quiz_complete` | пользователь прошёл мини-квиз |
| `lead_submit` | заявка успешно отправлена с формы |

Смотреть: **Отчёты → Конверсии**.

## Админка статистики (Telegram + Supabase)

Сейчас работает сразу:
- визиты отправляются в ваш Telegram (бот уже настроен в `js/main.js`)
- одно уведомление на сессию вкладки

Дополнительно (веб-админка):
- `admin.html` — вход и просмотр статистики через Supabase
- `js/main.js` — дублирует визиты в таблицу `visits` (если Supabase подключен)
- `js/supabase-config.js` — публичный конфиг Supabase

Скрытая ссылка на админку: точка `·` в футере сайта.

### Подключение Supabase (для веб-админки)

1. Создайте проект на https://supabase.com
2. В `js/supabase-config.js` вставьте `url` и `anonKey`
3. В SQL Editor выполните `supabase/schema.sql`
4. В Authentication -> Users создайте email+password пользователя
5. Откройте `admin.html` и войдите
